#### 1.1 登录页面

**需求分解:**

*   **核心功能:**
    *   **账号登录:**
        *   输入框：用户名/邮箱/手机号
        *   输入框：密码
        *   图形验证码
        *   按钮：“立即登录”
    *   **扫码登录:**
        *   二维码展示
        *   状态提示
*   **辅助功能:**
    *   链接：“注册12306账号”
    *   链接：“忘记密码？”
*   **异常处理:**
    *   登录失败提示（用户名/密码错误、验证码错误）
    *   账户锁定提示（多次尝试失败后）
    *   服务时间外提示

**场景描述:**

*   **场景一：用户使用账号密码成功登录**
    1.  用户进入登录页面。
    2.  输入正确的用户名和密码。
    3.  正确完成图形验证码。
    4.  点击“立即登录”按钮。
    5.  系统验证通过，页面跳转至12306首页或个人中心。

*   **场景二：用户登录失败（密码错误）**
    1.  用户进入登录页面。
    2.  输入错误的用户名或密码。
    3.  点击“立即登录”按钮。
    4.  页面提示“用户名或密码错误”，登录失败。

*   **场景三：用户登录失败（验证码错误）**
    1.  用户进入登录页面。
    2.  输入正确的用户名和密码。
    3.  输入错误的验证码。
    4.  点击“立即登录”按钮。
    5.  页面提示“验证码错误”，验证码刷新，登录失败。

*   **场景四：用户账户被锁定**
    1.  用户连续多次输入错误的密码。
    2.  系统锁定账户，并提示“账户已锁定，请稍后再试或找回密码”。

**需求分解:**

*   **核心功能:**
